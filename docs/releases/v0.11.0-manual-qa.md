# Koalafrog HQ v0.11.0 authenticated manual QA

Use an authenticated non-production test owner with an active reconciled workspace. Record IDs created during the run so duplicates and persistence can be checked precisely. Do not use production inventory.

## A. Beard Oil

- Open **Product Studio** and confirm Beard Oil is labelled `Formulation system: Simple liquid · operational`.
- Start **Design a Product**, choose target properties, and add a valid liquid base plus any supported additions.
- Confirm blocking findings prevent Formula handoff and explain what must change.
- Save the concept and record its URL and concept ID.
- Refresh the page and confirm name, targets, ingredients, roles, scent direction, and notes persist.
- Reach a valid 100% starting formula and create the Formula Draft once.
- Open the Formula and confirm Product, Formula, Draft Version, Formula Lines, percentages, roles, and phase labels are linked correctly.
- From the saved concept, create a Development Experiment. Save and approve a controlled variant through the normal lifecycle.
- Prepare a Lab Batch from the eligible Formula Version.
- Confirm planned ingredient quantities and the existing single-instruction process snapshot are present.
- Confirm no Inventory Movement exists until an explicit Lab allocation is committed.

## B. Beard Butter

- Open **Product Studio** and confirm Beard Butter is labelled `Formulation system: Anhydrous multi-phase · operational`.
- Create a Beard Butter concept and set target texture, firmness, melt behaviour, application feel, packaging intent, and climate notes.
- Add workspace Ingredients, assign each to a valid phase, and set percentages to exactly 100%.
- Confirm the phase cards and draft manufacturing steps contain no invented temperature limits.
- Add at least two development variants and record bounded 1–5 evaluation values.
- Save the concept, record its URL and concept ID, refresh, and confirm targets, phases, percentages, variants, and notes persist.
- Create the Formula Draft and confirm ordered phase definitions, Formula Line phase associations, and draft process steps are present.
- Create a Development Experiment and approve a controlled variant.
- Create the derived Formula branch and then prepare a Lab Batch.
- Confirm the Lab Batch contains ordered structured process steps with phase, critical-step, mixing, duration, and supported temperature metadata.
- Record actual temperature, operator note, deviations, packaging used, fill count, and final texture observations.
- Refresh and confirm all structured observations persist without altering the source Formula Version.
- Confirm no Inventory Lot or Inventory Movement was created by concept, Formula, Experiment, branch, or Lab preparation.

## C. Errors and idempotency

- Repeatedly activate the Product Studio Formula handoff for the same saved concept. Confirm the original Product, Formula, and Formula Version are reused and no duplicates appear.
- Repeat Development Formula-branch handoff with the same idempotency action. Confirm one branch exists.
- Repeat Lab handoff with the same idempotency action. Confirm one Lab Batch exists.
- In a controlled Draft test state, create a Formula Line referencing a nonexistent phase. Confirm an actionable unknown-phase error is shown and handoff is blocked.
- Verify duplicate phase codes, duplicate phase orders, duplicate process-step orders, inverted temperature ranges, negative durations, non-finite values, and totals outside the accepted 100% tolerance are rejected.
- Confirm Solid or stick, Emulsion, Water-based, Alcohol-based, Gel, and Powder are shown only as planned capabilities and cannot be started as Product Studio workflows.
- Attempt to open a concept using the wrong template URL and confirm it is not silently converted or saved under another product type.
- With a second authenticated owner, confirm the first owner’s concepts, formulas, experiments, Lab records, and backups are not visible.

## D. Backup and restore

- Open **Platform** from the subordinate **System** navigation group.
- Export a workspace JSON backup and inspect the envelope:
  - `applicationVersion` is `0.11.0`.
  - `workspace.sourceSchema` is `koalafrog-hq:workspace:v9`.
  - `format` is `koalafrog-backup-v1`.
- Run **Validate Backup** and confirm the new export is accepted.
- Validate a known legacy v9 backup whose `applicationVersion` is `0.1.0` and which lacks phase/process metadata; confirm it remains valid.
- Restore only into a controlled local test state using the established restore procedure.
- Confirm Beard Oil concepts and single-phase Formula records remain usable.
- Confirm Beard Butter concept analysis, Formula phases, manufacturing process, Formula Line phase associations, Lab process observations, and handoff links survive.
- Confirm workspace JSON restore does not create database/storage binary backups and does not infer or create inventory stock.
